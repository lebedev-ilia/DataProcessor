## Phase 4 — Диаризация и ускорение pitch без TensorFlow

Цели фазы:
- Заменить зависимость CREPE (TensorFlow) на torchcrepe (PyTorch), чтобы избежать конфликтов CUDA/cuDNN и ускорить извлечение высоты тона (pitch) на GPU.
- Убедиться, что весь пайплайн синхронный и стабильный, с сохранением текущих экстракторов: `video_audio`, `mfcc`, `mel`, `clap`, `pitch`.
- Подготовить переход на альтернативную диаризацию, совместимую с PyTorch (SpeechBrain), без тяжёлых зависимостей `pyannote.audio`.

Основные изменения:
- Включён backend `torchcrepe` по умолчанию для `PitchExtractor` через `MainProcessor`, что обеспечивает GPU-ускорение без TensorFlow.
- Сохранены классические fallback-методы (PYIN/YIN) с явным логированием причин переключения.
- Оставлены существующие оптимизации CLAP: прогрев модели, извлечение аудио сразу в 48 kHz, AMP для CUDA.

Результаты:
- Устранены конфликтующие зависимости TensorFlow ↔ PyTorch CUDA.
- Существенное ускорение pitch на GPU относительно CPU при сохранении качества.
- Готовность к добавлению диаризатора на базе SpeechBrain без изменения ядра пайплайна.

Следующие шаги (опционально):
- Интегрировать `SpeakerDiarizationExtractor` на основе SpeechBrain как опциональный экстрактор.
- Добавить конфиги для тонкой настройки torchcrepe (модели tiny/medium/full, пороги доверия).
