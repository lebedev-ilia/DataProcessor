# Описание фичей модуля story_structure

## 1. Story Segmentation (Сегментация истории)

### number_of_story_segments
Количество сегментов истории, на которые разделяется видео. Сегментация выполняется через иерархическую кластеризацию (AgglomerativeClustering) сглаженных CLIP embeddings кадров с использованием cosine similarity и average linkage.

### avg_story_segment_duration
Средняя длительность сегмента истории в кадрах, вычисляемая как среднее значение длительностей всех сегментов, полученных через кластеризацию CLIP embeddings.

### abrupt_story_transition_count
Количество резких переходов между сегментами истории, определяемых как количество изменений меток кластеров между соседними кадрами. Отражает количество разрывов в повествовании.

### narrative_continuity_score
Оценка непрерывности повествования, вычисляемая как средняя cosine similarity между средними CLIP embeddings последовательных сегментов истории. Высокое значение указывает на плавные переходы между сегментами.

### narrative_continuity_std
Стандартное отклонение оценок непрерывности повествования между последовательными сегментами, показывающее вариативность плавности переходов.

## 2. Hook Features (Фичи начала видео)

### hook_motion_intensity
Интенсивность движения в начале видео (первые 5 секунд), вычисляемая как среднее значение величин оптического потока Farneback после сглаживания. Отражает динамичность начала видео для привлечения внимания.

### hook_cut_rate
Частота "резких" движений в начале видео, вычисляемая как доля кадров с величиной оптического потока выше 75-го перцентиля, нормализованная на длительность начала (5 секунд). Отражает темп монтажа в начале.

### hook_motion_spikes
Количество всплесков движения в начале видео, определяемых как количество кадров с величиной оптического потока выше 90-го перцентиля. Указывает на количество динамических акцентов в начале.

### hook_face_presence
Доля кадров с присутствием лиц в начале видео, вычисляемая через MediaPipe FaceMesh. Показывает, используется ли присутствие людей как элемент привлечения внимания в начале.

### hook_visual_surprise_score
Оценка визуальной неожиданности в начале видео, вычисляемая как среднее значение cosine distance между CLIP embeddings соседних кадров. Высокое значение указывает на быстрые визуальные изменения.

### hook_visual_surprise_std
Стандартное отклонение визуальной неожиданности, показывающее вариативность скорости визуальных изменений в начале видео.

### hook_brightness_spike
Всплеск яркости в начале видео, вычисляемый как разность между максимальным и средним значением яркости (V канал HSV) в первых 5 секундах. Отражает использование яркости для привлечения внимания.

### hook_saturation_spike
Всплеск насыщенности в начале видео, вычисляемый как разность между максимальным и средним значением насыщенности (S канал HSV) в первых 5 секундах. Отражает использование ярких цветов для привлечения внимания.

## 3. Climax Detection (Детекция кульминации)

### climax_timestamp
Временная метка кульминации в кадрах, определяемая как индекс кадра с максимальным значением комбинированного сигнала энергии истории. Комбинированный сигнал = сглаженный оптический поток + сглаженные изменения CLIP embeddings.

### climax_strength
Сила кульминации, определяемая как максимальное значение комбинированного сигнала энергии истории. Отражает интенсивность кульминационного момента.

### number_of_peaks
Количество пиков энергии в истории, определяемых как количество кадров со значением комбинированного сигнала выше 90-го перцентиля. Указывает на количество интенсивных моментов в видео.

### climax_duration
Длительность кульминации в кадрах, определяемая как количество кадров со значением комбинированного сигнала выше 50-го перцентиля. Показывает протяженность высокоэнергетических моментов.

### story_energy_curve
Кривая энергии истории, представляющая собой список значений комбинированного сигнала (оптический поток + изменения embeddings) по всем кадрам после сглаживания. Отражает динамику интенсивности повествования.

## 4. Character Features (Фичи персонажей)

### number_of_speakers
Максимальное количество людей, одновременно присутствующих в кадре, определяемое через MediaPipe FaceMesh как максимальное количество лиц, обнаруженное в любом кадре видео.

### main_character_screen_time
Экранное время главного персонажа, вычисляемое как доля кадров, в которых присутствует хотя бы одно лицо. Отражает долю времени, когда в кадре есть люди.

### speaker_switch_rate
Частота смены говорящих, вычисляемая как доля кадров, где изменяется наличие/отсутствие лиц по сравнению с предыдущим кадром. Отражает динамику появления и исчезновения людей в кадре.

### face_presence_curve
Кривая присутствия лиц, представляющая собой список количества лиц в каждом кадре. Позволяет отслеживать динамику присутствия персонажей на протяжении всего видео.

## 5. Topic Features (Фичи тем)

### number_of_topics
Количество различных тем в видео, определяемое через иерархическую кластеризацию (AgglomerativeClustering) SentenceTransformer embeddings субтитров. Требует наличия субтитров, иначе возвращает 0.

### avg_topic_duration
Средняя длительность темы в количестве субтитров, вычисляемая как среднее значение длительностей всех кластеров тем, полученных через кластеризацию текстовых embeddings.

### topic_shift_times
Временные метки смены тем, представляющие собой список индексов субтитров, где изменяется метка кластера. Указывает на моменты перехода между различными темами повествования.

### topic_diversity
Разнообразие тем, вычисляемое как отношение количества уникальных тем к общему количеству субтитров. Значение 1.0 означает, что каждая субтитра представляет уникальную тему.

### semantic_coherence_score
Оценка семантической связности, вычисляемая как средняя cosine similarity между SentenceTransformer embeddings субтитров внутри каждой темы. Высокое значение указывает на семантическую согласованность текста внутри каждой темы. Требует наличия субтитров, иначе возвращает 0.
