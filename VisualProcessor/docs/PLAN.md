1. Должен быть модуль Segmenter (на уровне DataProcessor т.к segmenter сегментирует видео как для видео процессора так и для аудио процессора). На выходе Segmenter получаеться папка в которой лежит 2 папки, 1 - папка для VideoProcessor в которой лежат батчи видео фрэймов созданые frame_writer (реализован как пример в одном из модулей VideoProcessor), также там лежит файл метадаты самого видео и фрэймов + индексы фрэймов для каждого модуля. 2 папка - для AudioProcessor в которой лежит извлеченное сегментером аудио и метадата к нему.
2. (VideoProcessor) Для каждого модуля в файле README должен быть четко описан вход и выход модуля, а также используемы зависимости (библиотеки, модели) + модуль должен отвечать только за свой функционал и если ему требуються какие то дополнительные данные которе уже непосредственно не относяться к модулю, то это должно быть описано в README для того что бы выстроить взаимосвязи между модулями (например модуль similarity_metrics сам не анализирует видео а ждет что к нему на вход поступят выходы других модулей что бы он смог совершить свой анализ - это правильная работа).
Также очень важно что модули сами не работают с видео на прямую, каждый модуль если ему нужен доступ к кадрам обращаеться к папке с фрэймами (созданую Segmenter) и к метадате (там он ищет себя и то какие кадры ему отведены), далее с помощью FrameManager (реализация есть в одном из модулей VideoProcessor) он берет по индексу только выделенные кадры.
Также очень важно что модули не должны дублировать модели (например есть один модуль который работает с моделями YOLO. Этот модуль должен вычислять данные как для себя так и для всех других модуле которым нужна эта модель и он им просто отдает данные в чистом виде, что бы не использовать одну и ту же модель в разных модулях).
Модули вызываються через CLI (должен быть в каждом модуле) в main.py в VideoProcessor. Соответственно для такой реализации нужно четко отслеживать перемещение данных между модулями (например optical_flow провел анализ и должен в четком виде сохранить анализ в папку из которой потом модуль similarity metrics возьмет нужны данные для совего анализа, также важно расспределять сохраняемые данные на те которые являються конечными фичами и те которые сохранены для других модулей)
